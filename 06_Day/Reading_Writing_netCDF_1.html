Quick Install¶

pip install netCDF4

conda install -c conda-forge netCDF4

netcdf4-python is a Python interface to the netCDF C library.

Network Common Data Form (NetCDF): files are

  • self-describing (all documentation needed to work with data is included)

  • hierarchical data (organized in groups and variables) format.

  • Metadata and documentation of the data are included in the file itself

  • is convenient to store huge data.

  • The software is maintained by developers at Unidata, which is a subsidary of UCAR.

  • There are currently several versions of NetCDF in use:

    • NETCDF3_CLASSIC: was the original netcdf binary format, and was limited to file sizes less than 2 Gb.

    • NETCDF3_64BIT_OFFSET: was introduced in version 3.6.0 of the library, and extended the original binary format to allow for file sizes greater than 2 Gb.

    • NETCDF3_64BIT_DATA: is a new format that requires version 4.4.0 of the C library. It extends the NETCDF3_64BIT_OFFSET binary format to allow for unsigned/64 bit integer data types.

    • NETCDF4_CLASSIC: use the version 4 disk format (HDF5), but omits features not found in the version 3 API. They can also be read by HDF5 clients.

    • netCDF classic: is the original format and the most commonly used version, but it can only store up 2 Gigabits of data.

    • netCDF-4: is the latest generation of NetCDF. It has the data storage capabilities netCDF 64-bit and can handle all netCDF versions.

A NetCDF4 file has three parts:

  • variables and groups: main data product
  • dimensions: dimension of the variables
  • metadata: useful the information about variables.

Creating dimensions¶

  • createDimension method is used to create dimensions.

  • Every dimension has a name and a length.

  • Setting the dimension length to 0 or None makes it unlimited, so it can grow.

  • For NETCDF4 files, any variable's dimension can be unlimited.

  • For NETCDF4_CLASSIC and NETCDF3* files, only one per variable can be unlimited, and it must be the leftmost (slowest varying) dimension.

In [46]:
lat_dim = ncout.createDimension('lat', 73)     # latitude axis
lon_dim = ncout.createDimension('lon', 144)    # longitude axis
time_dim = ncout.createDimension('time', None) # unlimited axis (can be appended to).

Creating Variables¶

  • A variable has a name, a type, a shape, and data values.

  • The shape of a variable is specified by a tuple of dimension names.

  • A variable should also have some named attributes, such as 'units', that describe the data.

  • The createVariable method takes 3 mandatory args.

    • the 1st argument is the variable name (a string). This is used as the key to access the variable object from the variables dictionary.

    • the 2nd argument is the datatype (most numpy datatypes supported).

    • the third argument is a tuple containing the dimension names

    • the dimensions must be created first

lat.units = 'degrees_north'
lat.long_name = 'latitude'
